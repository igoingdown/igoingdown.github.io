家里突然断网？光猫 + 路由的分步重启排查手册

周日晚上家里网突然断了：手机、电脑、电视盒子全都连得上 WiFi，但一个外网都打不开。第一反应是”是不是欠费了 / 运营商挂了”，差点就打客服电话。结果什么都没动，只是按正确的顺序把光猫和几台路由断电、再按顺序开回来，网就通了。

听起来就是”重启一下”这种最没技术含量的操作，但这里面真正的关键是顺序——顺序错了，重启十遍也白搭。这篇把为什么要这么做、以及完整的操作步骤记下来。

一、先别急着拔电源：3 个前置判断

整套断电重启有点”重”（全屋设备都要重连），所以动手之前先花 1 分钟确认”该不该这么修”，免得白折腾。

  1. 是全网故障，还是单台设备的事？
    用手机连着同一个 WiFi 打开一个网页试试。

    • 手机也打不开 → 是整个网络的问题，往下走。
    • 只有电脑不行、手机正常 → 别动路由，问题在你这台电脑（DNS / 网卡 / 代理设置），先修本机。

  2. 先试最轻的：重置本机网络。
    如果只是本机可疑，先关掉 WiFi 再开 / 重启网卡 / ipconfig /flushdns（Windows）或 sudo dscacheutil -flushcache（Mac）。无效，再上整套重启。

  3. 看光猫的 LOS 灯——这一步能直接帮你甩锅。
    如果光猫上的 LOS 红灯在亮（或闪红），说明是光信号丢失，属于外线 / 光纤侧故障，不是你屋里设备能修的。这种情况别白忙活，直接打运营商报修。
    （LOS 不亮、PON 正常，才轮到自己重启。）

一句话：先分清”全网 vs 单机””内部 vs 外线”，再决定要不要全屋重启。

二、为什么”顺序”才是关键（不是玄学）

家里的网络是一条有上下游的链路：

下游设备依赖上游才能正常工作：

  • 光猫是总入口，负责把光信号转成网络，并和运营商建立 PPPoE 拨号会话。
  • 主路由要等光猫就绪，才能从上游拿到 WAN 地址、建立自己的路由表和 NAT。
  • 子路由 / AP 又要等主路由就绪，才能拿到 IP、挂到正确的网段上。

长时间运行后，这些设备里的会话、路由表、DHCP 租约、NAT / ARP 缓存可能错乱或溢出，于是出现”连得上 WiFi 但上不了网”——内网通、外网断，正是这种状态的典型表现。

所以两条铁律：

  • 断电要从上游往下、或整体一起断，并静置几分钟让缓存 / 会话彻底清掉（光猫和运营商那边的会话也需要时间释放）。
  • 上电必须严格从上游往下：上游没就绪就开下游，下游会拿不到地址、抓着错误状态启动，等于白重启——这就是”为什么顺序不能颠倒”。

三、断电操作（按顺序，断完静置）

  1. 断开光猫电源
  2. 断开主路由电源
  3. 断开所有子路由 / AP 电源
  4. 保持全部断电，静置 3~5 分钟——这一步别省，是给设备和运营商侧清空缓存 / 释放会话的时间

四、上电启动（顺序不能颠倒）

按上游 → 下游逐台开，每台等它稳定了再开下一台：

  1. 先开光猫，等 PON 灯常亮、且 LOS 不亮，表示已和上游对接成功（约 2~3 分钟）
  2. 光猫稳定后，再开主路由，等它系统加载完成、WAN / 联网指示灯正常
  3. 主路由正常后，依次开子路由 / AP
  4. 所有设备指示灯都稳定后，电脑 / 手机重连 WiFi，测试外网

关键点：**每一级都”等上一级稳了再开”**。心急一把全开，下游照样可能抓到”半就绪”的上游状态而失败。

五、光猫指示灯速查

灯 正常 异常含义
POWER 常亮 不亮 = 没供电 / 电源坏
PON 常亮 闪烁 = 正在和上游注册；长期不常亮 = 没对接上
LOS 不亮 红灯亮 / 闪 = 光信号丢失，外线故障，报运营商
LAN / 互联网 常亮或闪烁 不亮 = 下游没接好 / 没拿到地址

记住两个组合就够用：

  • PON 常亮 + LOS 不亮 = 光猫到运营商这段是好的，问题在你屋里 → 重启有救。
  • LOS 红 = 外线坏了 → 自己修不了，报修。

六、小结

这次”故障”最后没花一分钱、没等运营商，就靠按顺序重启解决了。能这么快定位，靠的是几个判断：

  1. 先用手机确认是全网问题，排除”只是我这台电脑”。
  2. 先看 LOS 灯分清内外，外线故障别自己瞎修。
  3. 断电静置给缓存 / 会话留出清空时间，不是拔了马上插。
  4. 上电严格按上游 → 下游、逐台等稳——顺序错了，重启就是做无用功。

说白了就一句话：家庭网络是一条有上下游的链路，重启要顺着链路的方向来。
